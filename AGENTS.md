# AGENTS.md

## Общие правила

- Отвечай и пиши комментарии на русском языке.
- Соблюдай существующую структуру проекта и минимизируй изменения вне задачи.
- Не меняй `dist/` вручную, если задача не требует обновления собранных артефактов; сначала правь `src/`, затем собирай проект.
- Перед завершением обязательно прогоняй проверки, которые затрагивают изменённые файлы.
- Если правишь тесты, держи имена в стиле `test_<что>_<условие>_<ожидание>`.

## Для TypeScript-проекта

- Импорты держи в начале файла.
- Публичные функции и методы типизируй явно, если это не мешает читаемости.
- Избегай лишних абстракций; выноси общий код только когда он реально повторяется.
- Не добавляй комментарии без необходимости; поясняй только неочевидные решения.

## Коммиты

- Используй конвенциональное наименование коммитов: `type(scope): summary`.
- `summary` пиши в повелительном наклонении, кратко и по существу.
- Предпочтительные типы: `feat`, `fix`, `docs`, `refactor`, `test`, `chore`.
- Для этого репозитория отдельные коммиты должны отражать одно логическое изменение.

## Проверки

- Перед коммитом запускай `npm run check`, если менялся код или тесты.
- Если изменён только документационный файл, можно ограничиться точечной проверкой содержимого и `git diff`.
